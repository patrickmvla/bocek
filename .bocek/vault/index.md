- [[discovery-preflight-mode-write-fails-under-sigpipe]] — type: discovery — `scripts/preflight.sh` silently fails to write `.bocek/mode` when its stdout is piped through a truncating consumer (`| head -8` etc.). `set -euo pipefail` + write-at-end design + ~50 prior echoes = SIGPIPE on the first echo after consumer closes its read end, abort before line 312 mode-write. Deterministic reproduction; observed during 2026-05-14 implementation session. Implementation bug; fix owed in `/design` (three options: move write to top vs trap SIGPIPE vs document-only).
- [[preflight-sigpipe-fix]] — type: decision — relocate `scripts/preflight.sh` mode-write + read-back verification block to immediately after `$PROJECT_ROOT` resolution (~line 47), before any echo to stdout. Eliminates the SIGPIPE failure mode structurally. Rejects Option 2 (trap SIGPIPE) because the invariant it preserves isn't user-observable, and Option 3 (document-only) per repeated doctrine-without-enforcement failure in this project. Queues S1: execute the relocation.
